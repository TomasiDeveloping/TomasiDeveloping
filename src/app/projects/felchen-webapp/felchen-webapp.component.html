Felchen Webapp

Die Felchen Webapp ist eine neue Version der Android Felchen App. Die neue Version wurde komplett als Webapp programmiert, die App ist eine normale Webseite und kann somit von jedem Gerät geöffnet werden, welches Internetzugang und einen Browser hat.

Der Vorteil liegt darin das die App nicht mehr nur an ein Android Gerät gebunden ist und auch Updates können viel schneller veröffentlicht werden

Der Nachteil ist das eine Webseite geöffnet werden muss und nicht einfach eine App gestartet werden kann.

Funktionen:

Die App ist mit einem Login geschützt. Ist der User nicht eingeloggt muss er sich als erstes einloggen oder Registrieren.

Nach dem erfolgreichen Login erhält der User ein Json Web Token welches 1 Tag gültig ist. Nach Ablauf des Tokens muss sich der User neu einloggen.

Auf der Startseite sieht der User seine eingetragenen Fänge, die Fänge sind standartmässig nach Fangdatum sortiert. Der User kann die Anzahl der Fänge pro Seite anpassen sowie nach Fangdatum sortieren. Mit Klick auf Detail zum Fang, kommt man zur Detail Ansicht des Fanges mit allen gespeicherten Daten

Mit dem Tab öffentliche Fänge sieht man alle Fänge welche User als öffentlich markiert haben.

Mit Klick auf Fang Detail in der Startseite, gelangt man zur Detail Ansicht des Fanges.

In der Detail Ansicht sieht der User alle Daten, welche zu diesem Fang gespeichert wurden. Die Daten lassen sich alle bearbeiten.

Der Fang lässt sich auch löschen.

Mit Klick auf Fang bearbeiten können alle Daten, welche gespeichert wurden zu diesem Fang bearbeitet werden.

Beim Bearbeiten werden keine aktuellen Wetterdaten geladen sowie beim Hinzufügen des Fanges. Alle Daten können manuell angepasst werden.

Im Menu Wetter werden dem User die aktuellen Wetterdaten für seinen Standort geladen. Die Standort Abfrage muss dem Gerät erlaubt werden, ansonsten werden keine Wetterdaten geladen.

Die Wetterdaten werden über eine API abgefragt und die Daten anschliessend dargestellt, diese Wetterdaten fliessen auch in einen neuen Fang ein sowie der Standort welches über das Gerät ermittelt wird.

Der User kann auch die Mondphasen anschauen der nächsten 7-Tage

Mit der 7 Tage Prognose kann der User die Wetterdaten für die nächsten Tage abfragen. Auch hier wird der Standort durch das Gerät ermittelt und eine Anfrage an eine Wetter API gesendet.

Die erhaltenen Daten werden für die nächsten 7-Tage aufbereitet und angezeigt.

Verwendete Technologien:

  • Angular
  • Bootstrap
  • .NET Core 5
  • MSSQL

Umsetzung:

Das Frontend wurde mit Angular programmiert und Bootstrap verwendet für die Darstellung

Für das Backend wurde eine RESTFull API mit .NET Core 5 in C# programmiert. Die REST wurde komplett asynchron programmiert

Die Daten werden in einer Microsoft SQL-Datenbank gespeichert.
