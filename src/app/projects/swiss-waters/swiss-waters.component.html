Schweizer Gewässer Daten

Die Idee zu dieser Applikation entstand wieder durch die Inspiration zum Angeln.

Ich stellte mir die Frage, ob ein Gewässer bereits die nötige Temperatur hatte, um einen ersten Nacht Angel Ausflug zu wagen.

Ich suchte im Internet wo und wie ich zu diesen Informationen kommen kann. Ich wurde auch fündig und konnte mir die Daten anschauen. Da kam mir die Idee, das andere Angler bestimmt auch an Gewässer Daten interessiert sind und solche Informationen gerne sehen würden.

Die Daten konnte ich mit etwas suchen beim Bundesamt für Umwelt finden. Ich kontaktierte das BAFU mit der Frage ob es möglich sei diese Daten zu beziehen.

Nach wenigen Tagen bekam ich Antwort und einen Zugang um die Daten direkt beim BAFU als XML abzurufen und ich begann die Applikation zu Entwickeln.

Startseite

Die Idee war eine Startseite zu Entwickeln wo alle Verfügbaren Station angezeigt werden sollen mit den Aktuellen Messdaten.

Es Entstand eine Card für jede einzelne Station mit den Verfügbaren Daten, die diese Station liefert.

Um es für den Benutzer etwas einfacher zu halten, gibt es eine Suchleiste, welche alle Stationen sortiert.

Mit dem Kantone Dropdown kann der Benutzer auch alle Gewässer zu den gewünschten Kantonen sortieren.

Mit Klick auf den Stern (Zu den Favoriten) können Stationen zu eigenen Favoriten hinzugefügt werden und auch nur nach diesen Sortiert werden.

Nicht alle Stationen bieten die gleichen Messdaten an.

Mit Klick auf Details kann für jede Station eine Detail Ansicht angezeigt werden.

In der Ansicht werden für alle Verfügbaren Messungen der Aktuelle Wert, Mittelwert letzten 24H, Maximalerwert letzten 24H und Minimalerwert letzten 24h angezeigt.

Verfügt die aktuelle Station über Temperatur Messwerte, werden alle Messungen der letzten 7 Tage in einem Diagramm angezeigt.

Die Applikation verfügt über eine öffentliche API.

Mit einem API Key können Benutzer Daten direkt als JSON über die API abrufen und in einer eigenen Applikation verarbeiten.

Verwendete Technologien:

  • Angular 13.3.0
  • Bootstrap 5
  • .NET 6
  • MSSQL
  • GRPC

Umsetzung:

Das Frontend wurde mit Angular programmiert und Bootstrap verwendet für die Darstellung

Für das Backend wurde eine RESTFull API mit .NET 6 in C# programmiert. Die REST wurde komplett asynchron programmiert

In die Applikation wurde ein GRPC Service implemnetiert für die Stationsdaten

Die Daten werden in einer Microsoft SQL-Datenbank gespeichert.
